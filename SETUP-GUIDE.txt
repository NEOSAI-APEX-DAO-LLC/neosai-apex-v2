STEP 2 — GITHUB SECRETS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Go to: github.com/officialelmalikpenguin/neosai-apex-v2
→ Settings → Secrets and Variables → Actions → New repository secret

Add ALL of these:

  CLOUDFLARE_API_TOKEN        (from dash.cloudflare.com → API Tokens)
  CLOUDFLARE_ACCOUNT_ID       (from Cloudflare dashboard right sidebar)
  ANTHROPIC_API_KEY           (from console.anthropic.com)
  STRIPE_SECRET_KEY           (from dashboard.stripe.com → Developers → API Keys)
  STRIPE_WEBHOOK_SECRET       (from Stripe → Webhooks → signing secret)
  SOVEREIGN_KEY               (make up a strong random string — this protects your worker)
  CLAUDE_CODE_ROUTINE_ID      (from claude.ai/code/routines → your routine → API trigger)
  CLAUDE_CODE_ROUTINE_TOKEN   (generated when you add API trigger to routine)

Frontend env (add to your .env or BuildAI settings):
  VITE_WORKER_URL=https://neosai-apex.officialelmalik.workers.dev
  VITE_SOVEREIGN_KEY=<same value as SOVEREIGN_KEY above>

In your terminal (Linux/Chromebook):

  cd ~/neosai-apex-v2
  
  # Copy all new files
  cp /path/to/downloaded/worker.js .
  cp /path/to/downloaded/wrangler.toml .
  mkdir -p .github/workflows
  cp /path/to/downloaded/deploy.yml .github/workflows/
  cp /path/to/downloaded/sovereign-api.js src/lib/
  
  # Commit and push — GitHub Actions auto-deploys
  git add .
  git commit -m "feat: unified sovereign engine — worker v3, stripe, aeon, vault"
  git push origin main

GitHub Actions will:
  1. Lint and validate
  2. Build frontend
  3. Deploy Cloudflare Worker
  4. Push all secrets to worker
  5. Run health check
  6. Fire A.E.O.N. routine to confirm deployment
